# GUIDE DÉPLOIEMENT PRODUCTION

## Backend - Railway (Recommandé)

1. **Créer un compte Railway.app**
   - Connectez votre repo GitHub
   - Créer nouveau project
   - Connecter la base de données PostgreSQL
   - Ajouter les variables d'environnement du .env

2. **Variables d'env à configurer**
   ```
   DB_HOST=... (Railway fourni)
   DB_NAME=gaspass_db
   DB_USER=... (Railway fourni)
   DB_PASSWORD=... (Railway fourni)
   PORT=5001
   FRONTEND_URL=https://gaspass.vercel.app
   JWT_SECRET=<long-random-string>
   STRIPE_SECRET_KEY=sk_live_...
   TELEGRAM_BOT_TOKEN=...
   TELEGRAM_ADMIN_GROUP_ID=-...
   SERVER_URL=https://gaspass-api.railway.app
   ```

3. **Deploy**
   - Railway détecte automatiquement Node.js
   - Lance `npm install` et `npm run db:sync`
   - Serveur live à: `https://<app-name>.railway.app`

## Frontend - Vercel

1. **Créer un compte Vercel**
   - Connecter repo GitHub
   - Configurer build command: `npm run build`
   - Output directory: `dist`

2. **Déployer**
   - Vercel déploie automatiquement à chaque push
   - URL: `https://gaspass.vercel.app`

## Base de Données - Railway / Neon

Railway fourni automatiquement PostgreSQL ou utiliser Neon:
- https://neon.tech
- Gratuit pour développement
- Production payant

## Configuration DNS

1. Acheter domaine (Namecheap, Cloudflare, etc)
2. Pointer DNS vers:
   - Frontend: Vercel nameservers
   - Backend: Railway URL ou custom domain

## Vérifier déploiement

```bash
# Backend
curl https://gaspass-api.railway.app/

# Frontend
https://gaspass.vercel.app/

# Admin
https://gaspass.vercel.app/admin/login
```

GitHub Actions (automatique avec Vercel/Railway)
- Test sur push
